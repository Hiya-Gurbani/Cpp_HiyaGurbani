Guidelines to be followed:
1. The identifiers (names of vatiable, functions, classes etc.) must be descriptive & 
    meaningful.
2. Always initialize variables to avoid undefined behavior. 
3. Preferable use const wherever possible.
4. The naming convention for variables and functions are snake_case 
    and camelCase. (In repo codes, camelCase would be preferred.)
5. The naming convention for classes would be PascalCase.
6. Function would follow Single Responsibility Principle.
7. Code should be modular and follow DRY (Don't Repeat Yourself) principle.
8. Use comment to explain why something is being done and not what.
9. Avoid using "using namespace std;" as it loads std to global scope & can 
    lead to conflicts & cause ambiguity.
10. Maintain code readability via formatting, indentation and consistent style.
